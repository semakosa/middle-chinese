# Middle Chinese Materials

[![GitHub Pages](https://img.shields.io/badge/GitHub%20Pages-live-2ea44f?logo=github)](https://semakosa.github.io/middle-chinese/)
[![Built with TshetUinh.js](https://img.shields.io/badge/built%20with-TshetUinh.js-555)](https://github.com/nk2028/tshet-uinh-js)

This repository hosts a range of materials developed by Xun Gong ([semakosa](https://github.com/semakosa)) on Middle Chinese phonology.

Most importantly, it provides a dataset of Early Middle Chinese (*Qièyùn* 切韻 system) character pronunciations and a web app that serves as a lookup tool for those pronunciations:

* [`character-readings.csv`](./character-readings.csv)
* https://semakosa.github.io/middle-chinese/

The transcription system provided here represents the full reconstruction of Early Middle Chinese developed in Wang & Gong (to appear). It builds on Xun Gong's earlier proposal, the “ʕayn theory” (Gong 2018), which identifies pharyngeality as the main phonetic feature characterizing Early Middle Chinese Division II. The main CSV and the web app provide a searchable implementation of this pharyngealization-based reconstruction of the *Qièyùn* system.

These serve as supplementary materials to:

> Tianheng Wang and Xun Gong. To appear. “Reconstructing pharyngealized vowels for *Qieyun* Division II in Middle Chinese.” *Diachronica*.
> [DOI: dia.25018.wan](https://doi.org/10.1075/dia.25018.wan)

## A dataset of Early Middle Chinese character readings

The main dataset is [`character-readings.csv`](./character-readings.csv). It gives character-level readings in a specific transcription system that implements the revised reconstruction of the *Qièyùn* system sketched in Appendix B of the article.

The CSV has four columns:

* `character`: Chinese character;
* `position`: EMC phonological position;
* `baxter`: Baxter-style transcription for reference; and
* `emc`: the Early Middle Chinese (*Qièyùn* system) pronunciation in the transcription system of this repository.

The dataset is machine-generated from the [tshet-uinh-data](https://github.com/nk2028/tshet-uinh-data) dataset of rhyme books in the *Qièyùn* tradition (*Guǎngyùn* 廣韻 supplemented by *Wángsān* 王三 data). The main dataset is generated by the derivation rules in [`rules/emc-wg2026-semakosa.js`](./rules/emc-wg2026-semakosa.js).

The underlying rhyme-book data and much of the computational infrastructure come from the nk2028 ecosystem, especially [TshetUinh.js](https://github.com/nk2028/tshet-uinh-js), [tshet-uinh-examples](https://github.com/nk2028/tshet-uinh-examples), and [Tshet-uinh Autoderiver](https://github.com/nk2028/tshet-uinh-autoderiver).

## A transcription system for Early Middle Chinese

The transcription system used for the `emc` column in this dataset is implemented in [`rules/emc-wg2026-semakosa.js`](./rules/emc-wg2026-semakosa.js). Its particular conventions will be documented separately. For now, the transcription system is defined by the derivation rules in that file.

The derivation rules originate from the `unt` system. They are revised from a larger set of rules in [`unt.js`](https://github.com/nk2028/tshet-uinh-examples/blob/main/unt.js) by [unt](https://github.com/untunt), which defines a parameterized range of transcription schemes that implement the revised reconstruction of Appendix B.

## Other materials

Less committed-to material is kept under [`testing/`](./testing). For now, it contains a larger version of the dataset with experimental Late Middle Chinese pronunciation data.

The web app and its published CSV copies are kept in [`webapp/`](./webapp).

## Project status

The main CSV and the web app are intended as supplementary materials to Wang & Gong (to appear). The readings are generated mechanically from the TshetUinh rhyme-book data and the local derivation rules. Some errors may still occur, but the upstream dataset has undergone years of iterative improvement and should therefore be mostly reliable.

As a matter of citation hygiene, the main CSV is mandated to not reflect scholarly positions that contradict Wang & Gong (to appear). If future improvements of this kind become desirable, they will be provided as successor datasets, possibly in a separate repository.

Please open an issue if you notice erroneous or unexpected data.

## Details

### Project layout

* `character-readings.csv`: the main dataset of Early Middle Chinese character readings.
* `rules/`: derivation rules for generating character readings from the rhyme-book data.
* `scripts/`: scripts for generating the dataset and other tools.
* `testing/`: more experimental materials, including the LMC rule under `testing/rules/`.
* `webapp/`: the web application and related materials.

### Regenerating the datasets

The generated CSV files can be refreshed by running [`scripts/deploy.sh`](./scripts/deploy.sh). This updates the root dataset and the experimental dataset under [`testing/`](./testing).
